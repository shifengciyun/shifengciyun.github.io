[CISCN 2023 初赛]烧烤摊儿

发现在gaming函数中存在将v5变量输入给全局name，并且没有对v5进行长度限制，有栈溢出
在.data里发现了name，可以考虑将/bin/sh写入.data段，因为是静态文件，其中没有libc和system，用ret2syscall构造execve(“/bin/sh”,NULL,NULL)get shell
确定程序中存在pop rax ；ret ，还要存在syscall

首先需要通过 j_strcpy_ifunc(&name, v5) 向 name 中写入 b’/bin/sh\x00’，并溢出 v5 构造 execve(“/bin/sh”, NULL, NULL) 执行

注意这里是将 b’/bin/sh\x00’ 写入到 name，所以只能一次性 get shell

如果分两次的话，例如：第一次写入 b’/bin/sh\x00’，第二次执行 execve(“/bin/sh”, NULL, NULL)，则在第二次中执行 j_strcpy_ifunc(&name, v5) 又会将 name 覆盖掉，导致 get shell 失败

法二：用mprotect

name在0x4E60F0
用vmmap查看发现0x4e6000-0x4e9000没有执行权限,先利用栈溢出执行mprotect函数，将其权限设为0x7，然后返回main函数，将shellcode写入name

使用open，read，write函数组合

got劫持

放进IDA可以看到有后门函数，但由于主函数是一直循环的，不能将地址直接覆盖为后门函数地址
那就是使用got劫持了，首先保护措施“Partial RELRO”，意味着GOT部分可写，即.got劫持可行
，通过格式化字符串漏洞的任意写将printf的got地址写成后门函数的地址，当再执行printf函数时就可以getshell

发现偏移为11

而栈的一行刚好能存储4B的数据（p32(printf_got)刚好4B），即格式控制符%11$n对应的要写入的地址就是printf_got
通过%c能控制输出字符的数量，控制p32(printf_got) + b’%’ + str(backdoor-4).encode()输出的字符数量刚好等于0x0804859B，与%n形成配合，最终使得printf_got地址处被写入了地址0x0804859B即backdoor的地址

但其实有函数直接将这些步骤执行了

fmtstr_payload函数帮我们干了所有的事情（除了定位输入位置，11）

定位好位置后，我们只需要输入一个字典元素比如Poc中的{printf_got: backdoor}

这个字典清晰地描述了我们的意图，即：“我想要在printf_got这个地址，写入backdoor这个值”

我们也可以同时提供多个写入目标，例如 {addr1: val1, addr2: val2}，pwntools会自动帮你生成一个能完成所有写入的payload
